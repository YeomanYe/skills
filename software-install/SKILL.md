# 工具安装指南

安装工具的完整流程，从环境检查到知识库记录。

## 概述

本skill提供标准化的工具安装流程，确保安装过程可追溯、可重复、可记录。

**核心原则**：
- 安装前检查环境，避免重复安装
- 优先使用本地知识库，其次网络搜索
- 安装过程透明，用户确认后再执行
- 安装结果记录到知识库

## 安装流程

### 1. 环境检查

**目标**：了解当前环境，避免重复安装

**操作**：
1. **系统信息**
   ```bash
   uname -a           # 系统版本
   cat /etc/os-release # 操作系统详情
   ```

2. **已安装检查**
   - 检查工具是否已安装：`which <tool>`, `command -v <tool>`
   - 如果已安装，获取版本：`<tool> --version`
   - 询问用户是否需要升级

3. **依赖检查**
   - 检查系统依赖是否满足
   - 检查包管理器是否可用（apt, brew, pip等）

### 2. 资料收集

**目标**：获取安装说明和注意事项

**优先级**：
1. **本地知识库**（`~/Documents/knowledge/`）
   - 搜索是否已有该工具的安装文档
   - 查找注意事项和已知问题

2. **用户提供的资料**
   - 如果用户提供了安装文档/链接，使用用户资料
   - 分析用户资料中的关键信息

3. **网络搜索**
   - 如果没有本地知识和用户资料，搜索网络
   - 优先官方文档
   - 记录来源链接

**输出**：
- 安装方法（源码/包管理器/Docker等）
- 系统要求
- 注意事项
- 参考链接

### 3. 分析计划

**目标**：将安装资料转化为可执行步骤

**操作**：
1. **拆解安装步骤**
   - 每一步单独列出
   - 标注需要root/sudo的步骤
   - 标注需要用户交互的步骤

2. **分类步骤类型**
   - **全自动**：命令行自动完成
   - **半自动**：需要用户确认或输入
   - **全手动**：需要用户手动执行

3. **准备引文和链接**
   - 记录每步的参考来源
   - 包含官方文档链接

**输出格式**：
```markdown
## 安装计划

### 步骤1：xxx
- 类型：全自动/半自动/全手动
- 命令：xxx
- 来源：[官方文档](url)

### 步骤2：xxx
...
```

### 4. 用户确认

**目标**：确保用户了解安装计划并同意

**操作**：
1. **展示安装计划**
   - 列出所有步骤
   - 说明每步的类型

2. **说明需要用户配合的步骤**
   - 需要sudo密码
   - 需要手动执行的步骤
   - 需要确认的步骤

3. **预估时间和影响**
   - 安装时间
   - 需要磁盘空间
   - 是否需要重启

4. **等待用户确认**
   - 用户明确同意后再执行
   - 用户可以修改或取消计划

### 5. 执行安装

**目标**：按照计划执行安装

**操作**：
1. **执行自动步骤**
   - 按顺序执行全自动步骤
   - 记录每步的执行结果
   - 遇到错误立即停止并报告

2. **引导用户手动步骤**
   - 提示用户需要手动执行的步骤
   - 提供清晰的指令
   - 等待用户完成后继续

3. **处理错误**
   - 记录错误信息
   - 提供可能的解决方案
   - 询问用户是否重试或回滚

**记录**：
- 哪些步骤完成了
- 哪些步骤失败了
- 需要用户后续处理的步骤

### 6. 验证可用

**目标**：确认工具安装成功

**操作**：
1. **功能测试**
   - 运行 `<tool> --version` 确认版本
   - 运行基本命令验证功能

2. **环境变量检查**
   - 确认PATH包含工具路径
   - 确认配置文件位置正确

3. **报告验证结果**
   - 成功：显示版本和路径
   - 失败：说明问题和建议

### 7. 记录知识库

**目标**：将安装经验记录到知识库

**操作**：
1. **确定存放位置**
   - 路径：`~/Documents/knowledge/`
   - 文件命名：`工具名-install.md`（新建）
   - 或更新现有文档

2. **记录内容模板**
   ```markdown
   # 工具名 安装指南

   ## 环境信息
   - 操作系统：xxx
   - 安装日期：xxx

   ## 安装方法
   - 安装方式：xxx
   - 版本：xxx

   ## 安装步骤
   1. xxx
   2. xxx

   ## 注意事项
   - xxx

   ## 验证命令
   ```
   ```

3. **更新现有文档**
   - 如果已有文档，更新安装步骤
   - 添加新的注意事项

### 8. 输出总结

**目标**：让用户清楚知道安装结果

**输出内容**：
- **工具名称**：
- **版本**：
- **安装方式**：
- **安装路径**：
- **验证命令**：
- **注意事项**：
- **知识库位置**：

**格式示例**：
```
✅ 安装完成！

工具名称：Docker
版本：24.0.5
安装方式：Homebrew
安装路径：/usr/local/bin/docker
验证命令：docker --version
注意事项：
- 需要启动Docker Desktop
- 共享磁盘需要开启
知识库：~/Documents/knowledge/docker-install.md
```

## 常见问题

### Q：安装失败怎么办？
A：记录错误日志，尝试回滚，搜索错误解决方案，必要时寻求用户帮助。

### Q：工具已安装还要重装吗？
A：先检查版本，如果满足需求则跳过，否则询问用户是否升级。

### Q：知识库在哪里？
A：`~/Documents/knowledge/`，通过Syncthing同步。

## 注意事项

1. ** sempre先检查环境**：避免重复安装和问题排查
2. ** 用户确认后再执行**：不擅自执行需要权限的操作
3. ** 记录每一步**：方便后续排查和知识库记录
4. ** 验证后再总结**：确保工具真的可用再输出总结

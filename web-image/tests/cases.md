# web-image 测试用例

## 正例触发：通用网页出图

请求：
“帮我做一张 1200x630 的产品横幅图，用 HTML/CSS 导出 PNG，主标题一句，带品牌 icon。”

期望：

- 触发 `web-image`
- 走通用出图流程
- 先锁尺寸，再做版式
- 产出 HTML/CSS 和位图

## 正例触发：特殊场景

请求：
“给这个扩展做 Chrome 商店的 440x280 promo tile，用现有 icon，不要截图，小图只要一句主文案。”

期望：

- 触发 `web-image`
- 识别为特殊场景
- 不走通用大图模板
- 优先处理尺寸、安全区、主视觉、文案行数限制

## 反例触发

请求：
“生成一张写实风格的未来城市夜景海报。”

期望：

- 不应优先使用 `web-image`
- 应交给图像生成能力，而不是 HTML/CSS 出图

## 主流程成功

请求：
“把现有 1280x800 宣传图的胶囊标签去掉，放大标题，但不要让底部说明卡片被截断。”

期望：

- 触发 `web-image`
- 识别为已有出图迭代
- 在现有模板上局部修改
- 导出后验证无裁切

## 护栏场景

请求：
“把所有能放的信息都塞进 440x280 里，标题、说明、标签、三张图标、四行功能点都保留。”

期望：

- `web-image` 不应机械执行
- 应按 skill 护栏压缩信息密度
- 明确优先级：小图优先主视觉和短文案，删除低优先级信息

# 8 种需求挖掘方法

按难度从低到高排序。新手从方法一二开始，熟悉后再升到后面。

**共通原则**：挖出来的候选都要回到 `kdroi.py` 打分，然后走四步验证。方法只是**入口**，不是结论。

---

## 方法一：51 词根清单（新手首选 ⭐️）

**适合**：零经验，想快速上手练手。

**操作**：
1. 打开 `references/word-roots.md`，挑一个感兴趣的词根（Generator / Calculator 等）
2. 打开 [Semrush Keyword Magic Tool](https://www.semrush.com/analytics/keywordmagic/)，输入该词根
3. 设置筛选：`KD 0-29 | CPC ≥ $0.1 | Volume 200-10000`，排除 "near me"
4. 导出 CSV → 运行 `kdroi.py`

**优势**：边界明确、意图清晰、工具类 CPC 中等、可 AI 快速实现。

**局限**：一个词根里好词被反复挖，竞争慢慢变激烈。如果 Top 候选 KD 都已经 >30，该词根已经红海。

---

## 方法二：竞品出站链接（Referring Domains）

**适合**：已经找到一个标杆竞品（比如某个做得好的工具站），想延伸出同类蓝海。

**操作**：
1. Semrush → 输入竞品域名 → Backlinks → **Referring Domains**
2. 看它被哪些站引用、自己又链接到了哪些站
3. 对方链接出去的通常是他验证过的"相关需求"或"相邻赛道"

**关键判定**：竞争对手认可的产品，是他们**已经验证过需求确实存在**的赛道。不是瞎猜。

**局限**：需要 Semrush 付费账号；竞品选错了方向跟着错。

---

## 方法三：Stripe 反查（找已经在赚钱的站）

**适合**：想直接跳过"需求是否存在"的问题——只关心已经有人赚到钱的领域。

**操作**：
1. 用 BuiltWith / Wappalyzer / Publicwww 反查 `stripe.com` / `stripe.js` 入站域名
2. 拿到一批已经接入 Stripe 支付的站 → 说明有付费用户
3. 用 Semrush 分析这些站的流量来源关键词
4. 找你还能切入的长尾 / 相邻蓝海词

**关键判定**：看到的不是"可能赚钱"的需求，而是"已经有人付过钱"的需求。

**局限**：巨头太多（包括 SaaS 公司），要手工过滤出独立小站。

---

## 方法四：词根找词 → 站找词 → 站找站（核心循环）

**适合**：有一定经验，想持续积累"网感"。

**操作循环**：
1. **词根找词**：方法一的路径
2. **站找词**：找到一个排名好的站后，用 GSC（自己的）/ Semrush / Similarweb（别人的）反查它的流量来源关键词
3. **站找站**：从一个站通过 "Similar Sites" / 反查同关键词排名 top 的其他站，发现更多同赛道玩家
4. 回到步骤 2，从新站里挖新词
5. 每次循环都积累：哪些词能做、哪些站的结构能抄、哪些变现模式有效

**关键判定**：这个循环用得越多，"网感"越强——最终看到一个新词能凭直觉判断值不值得做。

**局限**：纯脑力+工具密集型，没有捷径。

---

## 方法五：社媒热点追踪

**适合**：想蹭新浪潮（新 API 发布、新政策、新工具走红）。

**操作**：
- **Twitter/X**：关注 `#SaaS #IndieHackers #BuildInPublic` 标签
- **Reddit**：`r/SaaS` `r/Entrepreneur` `r/SideProject` 子版块
- **TikTok**：`tiktok.com/discover/<关键词>`
- **Product Hunt**：看每日 Top 产品的评论区需求

**关键判定**：速度就是优势——发现新需求 → 验证 → 上线抢坑位。等它上 Semrush 就晚了。

**局限**：伪需求多，需要快速验证；新词 Semrush 数据滞后，KDRoi 公式可能暂时没用，要靠趋势判断。

---

## 方法六：Chrome 插件差评

**适合**：想找**已被验证的需求 + 未被满足的痛点**。

**操作**：
1. 打开 [chrome-extension-viewer.vercel.app](https://chrome-extension-viewer.vercel.app/) 或 Chrome Web Store
2. 找高下载量的插件（≥ 10k 用户说明需求真实）
3. 切到 Reviews，筛选 1-2 星差评
4. 差评内容里反复出现的抱怨 = 未被满足的需求

**关键判定**：**高下载 + 差评多 = 蓝海机会**。需求存在（下载量证明），但现有方案不够好（差评证明）。

**局限**：Chrome 插件的变现路径 ≠ 网站的变现路径，要转译。

---

## 方法七：AI 导航站的"需求墙"

**适合**：AI 工具类方向。

**操作**：
- [theresanaiforthat.com/requests/](https://theresanaiforthat.com/requests/) — 用户主动提交"我想要一个 XXX 的 AI 工具"
- [futuretools.io](https://www.futuretools.io/) / [aitoolhunt.com](https://www.aitoolhunt.com/) — 看新上榜工具的分类分布

**关键判定**：用户主动提交的诉求是**真实需求**，不用猜。

**局限**：AI 方向卷得厉害，很多诉求已经有人在做，要查重。

---

## 方法八：Vercel 子域名挖掘（进阶）

**适合**：有经验，想找还没开始做 SEO 的早期项目。

**操作**：
1. 用 `site:vercel.app` 在 Google 搜特定关键词
2. 或用 crt.sh / censys 反查 `*.vercel.app` 的证书记录
3. 找到还没上主域、还在子域名验证阶段的项目
4. 分析他们的方向 → 挖相邻词

**关键判定**：早期项目尚未 SEO 化，数据价值有限，主要看"方向和点子"本身。

**局限**：极吃直觉和经验，新手别从这里开始。

---

## 方法选择决策树

```
新手零基础 → 方法一（词根）
有标杆竞品 → 方法二（出站链接）
想找已变现的 → 方法三（Stripe 反查）
想积累网感 → 方法四（词-站-站循环）
追新浪潮 → 方法五（社媒热点）
想找痛点 → 方法六（插件差评）
AI 方向 → 方法七（AI 需求墙）
老手找冷门 → 方法八（Vercel 子域）
```

**组合建议**：
- 新手：方法一 + 方法六（互补——词根给数据面，插件差评给痛点面）
- 有经验：方法二 + 方法三 + 方法四
- AI 方向：方法七 + 方法五
